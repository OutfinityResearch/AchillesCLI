# spec-help

Explain how the GAMP-style specification stack works.

## Summary
- Provides a quick refresher on the URS → FS/NFS → DS → Test lifecycle.
- Highlights key concepts, lifecycle steps, and best practices to keep specs healthy.
- Works even without an LLM by falling back to a curated help message.

## Instructions
- Keep guidance actionable and concise.
- Emphasise traceability, per-requirement DS documents, and embedded tests.
- Encourage users to pair this overview with review/mentor skills when planning changes.
